# Tips for using session-defaults

- Always create and enter your new project folder before running the setup script.
- Use the provided curl | bash command to fetch and run setup.sh directly in your new folder.
- The setup script will use the current folder name as the project name automatically.
- After setup, run the 'start session' command to initialize your session.
- For repeated use, consider creating a shell alias for the setup command.

- All workflow tips and session notes are now stored in the memory folder (memory/tips.md and memory/session-notes.md) for better organization and future reuse.
- The old tips.md in the project root was removed to avoid duplication and keep everything in one place.
